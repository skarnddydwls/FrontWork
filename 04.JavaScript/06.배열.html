배열

- 변수 선언 시 별도의 자료형 지정하지 않음
- 어떤 자료형의 값들이든 모두 하나의 배열 공간에 담을 수 있음
- 크기의 제약이 없음
- 존재하지 않는 인덱스를 참조해도 오류나지 않음 (undefined리턴)



* 배열의 선언



* 배열 관련 메소드

1) 배열.indexOf(찾고자하는 요소): 배열에서 해당 요소가 있는 index 반환



2) 배열.concat(배열, 배열, ...): 배열 여러개를 결합하고자 할 때 사용



3) 배열.reverse(): 배열에 담긴 값들을 역순으로 바꿔주는 메소드

- 원본 배열에 영향을 끼치는 메소드



4) 배열.sort: 배열안에 담긴 값들을 오름사춘으로 정렬해주는 메소드

- 원본 배열에 영향을 끼치는 메소드



5-1) 배열.push(추가할요소): 배열의 맨 뒤에 요소 추가시키고 배열의 크기 반환
5-2) 배열.pop(): 배열의 맨 뒤 요소를 제거하고 제거된 요소 반환



6-1) 배열.unshift(추가할요소): 배열의 맨 뒤에 요소 추가시키고 배열의 크기 반환
6-2) 배열.shift(): 배열의 맨 뒤 요소를 제거하고 제거된 요소 반환



7-1) 배열.slice(시작 인덱스,끝 인덱스): 배열로부터 시작 인덱스부터 끝 인덱스전까지 추출해주는 메소드
7-2) 배열.splice(시작 인덱스,끝 인덱스[, 추가할 값]): 배열의 요소를 추출하여 제거 및 추가, 제거한 요소 반환



8) 배열.toString() / 배열.join([구분자]): 배열의 각 인덱스에 담긴 값들을 하나의 '문자열'로 합쳐서 반환



9) 반복처리(for문, for ..in문, for ..of문, foreEach)
